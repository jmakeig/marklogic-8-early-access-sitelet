Native JSON

What it is

JSON is rapidly becoming the data format of choice for many use cases; particularly those involving data serialization, as opposed to content markup. Because XML and JSON each have areas where they excel, it is important that developers are able to choose the most appropriate representation for their use case without giving up on any of the core capabilities that MarkLogic provides.

The addition of native JSON support does just that – it provides full-fledged JSON support alongside XML, text, and binary. Developers can use the right format for the right task without giving up ACID transactions, rich query, or any of the other capabilities they have come to rely on. Ultimately this makes developers more productive, simplifies applications, and reduces overhead.

Early Access 3, November 2014

Early Access 2, August 2014

Early Access 2 builds on the initial Native JSON feature set from EA1 (see below) with:

  • JSON serializations of all query types (cts.andQuery, cts.propertyValueQuery, etc.), except for geospatial queries, reverse queries, bi-temporal, and similar queries. These are scheduled for Early Access 3. Use the serializations to persist queries and re-instantiate them using the cts.query constructor in JavaScript or cts:query in XQuery.
  • JavaScript built-in functions that take permissions as input or return permissions as output now use JavaScript objects to represent the permissions, for example xdmp.documentInsert or xdmp.documentGetPermissions.
  • Options parameters in most JavaScript built-in functions are now specified as JavaScript objects, for example, the third parameter of xdmp.eval.
  • cts:walk and cts:highlight in XQuery on JSON nodes. cts.walk on JSON nodes in JavaScript it scheduled for Early Access 3.
  • Just like you’d use a cts:element-reference to identify a range index on an XML element in a cts.values call or as part of a cts.indexOrder specification, you can now create references to range indexes on JSON properties with cts.jsonPropertyReference in JavaScript or cts:json-property-reference in XQuery.
  • Field word queries now support JSON documents. Field value and field range queries on JSON documents will be supported in Early Access 3.

Early Access 1, June 2014

Early Access 1 introduced the following JSON capabilities:

  • Full-text indexes, including advanced language stemming, tokenization and de-compounding for JSON strings.
  • Document-level role-based access controls.
  • Multi-statement and multi-document ACID transactions.
  • XPath selection and projection over JSON.
  • New JSON types for XQuery applications.
  • Create update, and delete native JSON documents from XQuery
  • Query JSON properties and values using new and existing built-in functions (for more info see details in deeper dive)

Important Concepts

Nodes

JSON documents are represented as hierarchies of nodes. Each node has a type and a value. For example, the following document is illustrated as a tree of nodes below.

{ 
 "name": "Oliver", 
 "scores": [88, 67, 73],
 "isActive": true, 
 "affiliation": null 
}

The JSON data model defines six data types. Each is represented as a node type in MarkLogic. Additionally, MarkLogic wraps JSON nodes in a Document node in and out of the database.

  • Object: Key-value pairs
  • Array: Ordered collections
  • Text: Character data
  • Number: Integer or decimal values
  • Boolean: true or false
  • Null: Absence of another type, but still defined
  • Document: Top-level wrapper for documents in the database.

mlcp

JSON is now a document format, just like XML, text and binary. JSON documents support permissions,collections, quality in the same way that is possible with other document formats. You can ingest JSON documents through the Java and Node.js Client APIs, mlcp, Server-Side JavaScript, or XQuery. Here is a command line example to ingest JSON documents with mlcp:

./mlcp import -host localhost -port 8011 -username admin -password admin -input_file_path /space2/jsondata/

Compressed archives are also supported, for example,

./mlcp import -host localhost -port 8011 -username admin -password admin -input_file_path /space2/jsondata-zipped/ -input_compressed true -input_compression_codec zip
