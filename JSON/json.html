Native JSON

Where to focus

Early Access 2 builds on the initial Native JSON feature set from EA1 (see below) with:

  • JSON serializations of all query types (cts.and-query, cts.property-value-query, etc.), except for geospatial queries, reverse queries and bi-temporal queries. These are scheduled for Early Access 3. Use the serializations to persist queries and re-instantiate them using the cts.query constructor in JavaScript or cts:query in XQuery.
  • The ability to specify permissions and options parameters on most built-ins as JSON, for example xdmp.eval in JavaScript or xdmp:eval in XQuery.
  • cts:walk and cts:highlight in XQuery on JSON nodes. cts.walk on JSON nodes in JavaScript it scheduled for Early Access 3.
  • Just like youâ€™d use a cts:element-reference to identify a range index on an XML element in a cts.values call or as part of a cts.order specification, you can now create references to range indexes on JSON properties with cts.jsonPropertyReference in JavaScript or cts:json-property-reference in XQuery.

Early Access 1 introduced the following JSON capabilities:

  • Full-text indexes, including advanced language stemming, tokenization and de-compounding for JSON strings.
  • Document-level role-based access controls.
  • Multi-statement and multi-document ACID transactions.
  • XPath selection and projection over JSON.
  • New JSON types for XQuery applications.
  • Create update, and delete native JSON documents from XQuery
  • Query JSON properties and values using new and existing built-in functions (for more info see details in deeper dive)
